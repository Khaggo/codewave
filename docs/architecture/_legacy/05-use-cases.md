# Use Cases

This document focuses on actor-to-capability mapping.

## Customer

- register and log in
- manage profile
- manage vehicles
- create service bookings
- track booking status
- view vehicle lifecycle timeline
- submit insurance inquiries
- browse products
- place invoice-based orders
- check purchase history
- use the rule-based chatbot
- track back-job status when a return case exists

## Admin / Staff / Manager

- manage bookings and schedules
- record vehicle inspections
- monitor job progress
- open and manage back jobs
- track insurance inquiries and documents
- manage customer and vehicle records
- review loyalty and reward activity
- monitor dashboard analytics
- monitor inventory and invoice payment status summaries

## Technician

- view assigned work
- update job progress
- contribute operational findings where allowed
- participate in completion and return workflows

## Notification Service

- send booking reminders
- send service schedule reminders
- send renewal alerts
- send operational status messages

## E-Commerce Service

- expose product and category data
- manage cart and invoice checkout
- manage orders and invoice payments
- publish commerce events for downstream consumers

## Key Visibility Rules

- customers should not automatically see internal-only notes
- inspection verification should remain staff-controlled
- back-job classification should remain an operational decision
