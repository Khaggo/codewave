============================================================
PART 1 - START THE SYSTEM
============================================================

1. Start Docker services
Open a terminal:

cd D:\mainprojects\codewave\backend
docker compose up -d

2. Push the database schema
Run:

npm run db:push

3. Start the backend
Run:

npm run dev:main

Expected result:
- Backend runs on port 3000.
- Health check should load at:
  http://127.0.0.1:3000/api/health

4. Start the web app
Open a second terminal:

cd D:\mainprojects\codewave\frontend
npm run dev -- --port 3002

Expected result:
- Web opens at:
  http://127.0.0.1:3002

5. Start the mobile app for Expo Go
Open a third terminal:

cd D:\mainprojects\codewave\mobile
npx expo start --lan --clear

Expected result:
- Expo shows a QR code.
- Phone and computer must be on the same Wi-Fi.
- mobile/.env.local should point to the computer LAN IP, for example:
  EXPO_PUBLIC_API_BASE_URL=http://192.168.254.109:3000

6. Start mobile in Expo web if testing in browser
Run from the mobile folder:

npx expo start --web --clear

Expected result:
- Expo web opens in the browser.
- If testing from the same computer, mobile/.env.local can use:
  EXPO_PUBLIC_API_BASE_URL=http://127.0.0.1:3000

PART 5 - COMMON ISSUES AND WHAT THEY MEAN
============================================================

Issue: Mobile cannot connect to backend in Expo Go.
Likely cause:
- mobile/.env.local is using 127.0.0.1 instead of the computer LAN IP.
Fix:
- Use the computer Wi-Fi IP:
  EXPO_PUBLIC_API_BASE_URL=http://YOUR_COMPUTER_LAN_IP:3000

Issue: Web cannot load backend data.
Likely cause:
- Backend is not running on port 3000.
Fix:
- Start backend and open:
  http://127.0.0.1:3000/api/health

Issue: A new backend endpoint does not exist.
Likely cause:
- Backend server was not restarted after code changes.
Fix:
- Stop and restart npm run dev:main.

Issue: Staff account was created but cannot log in.
Likely cause:
- The backend is running old code, the account is inactive, or the wrong generated email/password was used.
Fix:
- Restart backend.
- Create a new account from Staff Accounts.
- Use the generated email and the exact password entered during creation.

Issue: Mobile booking appears on phone but not web schedule.
Likely cause:
- Wrong selected date on web, backend mismatch, or mobile is pointing to a different backend URL.
Fix:
- Confirm mobile EXPO_PUBLIC_API_BASE_URL points to the same backend.
- Select the exact booking date in web Booking Schedule.

Issue: Mobile layout shifts on another phone.
Likely cause:
- The screen width triggers a different compact layout.
Fix:
- Test on small, baseline, and larger phone sizes.
- Prioritize checking Book, Home, Garage, Shop, Insurance, Rewards, and bottom navigation.
